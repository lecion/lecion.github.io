JMM之Volatile

By Bubble

2014-12-26 更新日期:2015-09-13

文章目錄

很长一段时间认为，volatile修饰的变量具有原子性。今天看JMM时发现错得很惨- -

public class Main {
    long count = 0;
    public static void main(String[] args) {
        new Main().start();
    }

    public void start() {
        for (int i = 0; i &lt; 1000; i++) {
            new Thread(new Runnable() {
                @Override
                public void run() {
                    inc();
                }
            }).start();
        }
        try {
            Thread.sleep(2000);
            System.out.println(count);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }

    public void inc() {
        this.count++;
    }
}
`</pre>

为保证看到最后结果，加了个sleep，保证所有线程跑完。。 然后呢，结果是随机的，可能是995，也可能是998，还可能是..... 这说明volatile是不能让修饰的变量具有原子性的。它只能让一个变量的`读`or`写`让所有线程可见。 作用等效于

<pre>`public synchronized void read() {
    return this.count;
}

而不能用于复合操作count++。

值得注意的是，count 是 long 型。占64位，总线调度传输的数据一次是32位，所以对java中long和double这样的64位数据，是分成两个32位进行处理的，也就是对count的传送，也不是一个原子操作。但是加了volatile之后，便可以保证对于64位类型的变量也是原子操作了。
